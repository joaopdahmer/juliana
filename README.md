## Publicar no GitHub Pages

### 1. Criar o repositório

No GitHub:

1. Clique em `+`.
2. Clique em `New repository`.
3. Use um nome como `amor`.
4. Marque como `Public`.
5. Não marque a criação automática de README.
6. Clique em `Create repository`.

### 2. Enviar os arquivos

Descompacte o ZIP e abra a pasta `amor-joao-juliana`.

Na página do repositório:

1. Clique em `Add file`.
2. Clique em `Upload files`.
3. Arraste **o conteúdo de dentro da pasta**, incluindo:
   - `index.html`
   - `style.css`
   - `script.js`
   - `.nojekyll`
   - `README.md`
   - pasta `assets`
4. Clique em `Commit changes`.

O arquivo `index.html` deve aparecer na página principal do repositório. Ele não pode ficar preso dentro de outra pasta.

### 3. Ativar o site

1. Abra `Settings`.
2. Entre em `Pages`.
3. Em `Source`, selecione `Deploy from a branch`.
4. Em `Branch`, selecione `main`.
5. Em pasta, selecione `/ (root)`.
6. Clique em `Save`.

O endereço ficará parecido com:

```text
https://SEU-USUARIO.github.io/amor/
```

A primeira publicação pode levar alguns minutos.

## Editar depois

- Textos: `index.html`
- Cores e aparência: `style.css`
- Data do contador: `script.js`
- Fotografias: pasta `assets`

## Observação de privacidade

O GitHub Pages deixa o site acessível publicamente. As fotografias e os textos enviados ao repositório poderão ser vistos por qualquer pessoa que possua ou encontre o endereço.
